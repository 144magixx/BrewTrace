# 项目文档索引

本文档约定项目文档的归档位置，避免文档散落在项目根目录。

## 目录约定

- `docs/prd/`：产品需求文档、需求迭代记录。
- `docs/architecture/`：技术架构、Agent 架构、模块设计。
- `docs/research/`：外部平台、竞品、技术可行性调研。
- `docs/learn/`：学习文档、技术选型原因、关键设计讲解。
- `docs/meeting-notes/`：讨论纪要、需求对齐记录。

## 维护原则

- 新文档优先放入对应子目录，不直接放在项目根目录。
- 文档文件名使用小写英文和连字符，版本号写入文件名或文档标题。
- 涉及未验证信息时，应明确标记为“待调研”或“待确认”。
- PRD、架构、调研文档分开放置，避免一个文档承担所有职责。
