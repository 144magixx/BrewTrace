# 小红书真实发布人工验证记录模板 v0.1

## 已确认事实

- 自动化测试不依赖真实小红书账号、验证码、浏览器页面或真实发布动作。
- 后端发布流程要求发布包确认和发布页预览后二次确认。

## 验证记录

| 项目 | 记录 |
|---|---|
| 验证日期 | 待填写 |
| 验证账号 | 仅记录账号摘要，不记录 Cookie、Token 或密码 |
| 登录状态 | 待填写 |
| 发布包 ID | 待填写 |
| 填写发布页结果 | 待填写 |
| 是否完成预览后二次确认 | 待填写 |
| 是否公开发布 | 待填写 |
| 失败原因 | 待填写 |

## 安全检查

- [ ] 未二次确认时没有点击公开发布。
- [ ] 外部参考均标明来源，未写成用户事实。
- [ ] 日志、文档和 Agent 轨迹中没有完整 API Key、Cookie、Authorization Header 或 Session Token。
- [ ] 登录受阻时发布包保留，并返回用户可处理路径。
