# 风味联想体系调研 v0.1

## 1. 调研目标

设计一套适合咖啡品鉴创作 Agent 的风味联想体系。它应能帮助用户把模糊描述扩展成更具体、更有画面感的表达，同时不强制绑定专业杯测体系。

## 2. 参考资料

参考资料：

- SCA Coffee Taster's Flavor Wheel：https://sca.coffee/research/coffee-tasters-flavor-wheel
- World Coffee Research Sensory Lexicon：https://worldcoffeeresearch.org/resources/sensory-lexicon

SCA 风味轮适合作为专业分类参考。WCR Sensory Lexicon 适合作为感官描述的底层词库参考。

产品原则：

- 参考专业体系，但不被专业体系限制。
- 风味联想服务创作表达，而不是替用户做专业认证。
- 用户确认前，所有联想都只能作为候选。

## 3. 第一版风味层级

建议第一版使用三层结构：

```text
大类 -> 子类 -> 具体风味
```

示例：

- 果香 -> 柑橘 -> 柠檬、青柠、甜橙、血橙、葡萄柚、蜜柑、柚子。
- 果香 -> 浆果 -> 草莓、覆盆子、蓝莓、黑加仑。
- 花香 -> 白花 -> 茉莉、橙花、栀子。
- 甜感 -> 糖类 -> 蜂蜜、焦糖、红糖、枫糖。
- 茶感 -> 茶类 -> 红茶、乌龙、绿茶、伯爵茶。
- 坚果可可 -> 坚果 -> 杏仁、榛子、核桃。
- 发酵酒感 -> 酒香 -> 红酒、白兰地、米酒、果酒。
- 负面风味 -> 瑕疵 -> 纸板、木头、土腥、橡胶、药感。

## 4. 温度段设计

每次记录按三个温度段描述：

- 高温：更容易感受到香气、挥发性气味、明亮酸质。
- 中温：甜感、果汁感、层次和平衡更容易展开。
- 低温：余韵、发酵感、苦韵、干净度缺陷更容易显现。

注意：这是产品化经验假设，不应当作为绝对规律。

## 5. 香气与味道区分

每个温度段下区分：

- 香气：闻到的气味和鼻后香气。
- 味道：入口后的酸甜苦、口感、余韵和风味联想。

示例：

```text
高温 / 香气：橙花、柑橘皮
高温 / 味道：青柠酸质、轻微茶感
中温 / 香气：蜂蜜、红茶
中温 / 味道：甜橙、黄桃、红茶尾韵
低温 / 香气：果脯
低温 / 味道：葡萄柚皮、轻微清苦
```

## 6. 联想生成策略

输入一个词时，系统生成候选项。

候选项包含：

- 风味名词。
- 描述短语。
- 适合出现的温度段。
- 更偏香气还是味道。
- 正面或负面倾向。
- 置信度或推荐理由。

示例：

```json
{
  "input": "柑橘",
  "suggestions": [
    {
      "name": "甜橙",
      "description": "圆润、亲和、甜感明显",
      "stage": "中温",
      "senseType": "味道",
      "polarity": "正面"
    }
  ]
}
```

## 7. 正面与负面风味

正面风味不等于高级，负面风味也不等于完全不可用。系统应帮助用户描述真实体验。

正面示例：

- 花香、果香、蜂蜜、红茶、坚果、可可、果汁感。

负面示例：

- 纸板、木质、烟感、土腥、橡胶、药感、尖酸、空洞、涩感。

表达时要区分：

- 风味本身。
- 强度。
- 用户主观喜好。
- 是否影响整体平衡。

## 8. 风味联想与文案关系

风味联想只负责打开候选空间，不直接决定最终文案。

最终文案生成时需要检查：

- 用户是否确认该风味。
- 该风味属于香气还是味道。
- 出现在哪个温度段。
- 是否与感官评分冲突。
- 是否与历史文案重复。

## 9. 学习重点

学习这个模块时，重点关注：

- 词库、规则和大模型联想如何结合。
- 如何把模糊输入转成结构化候选。
- 如何避免模型过度发挥。
- 如何让用户选择，而不是替用户下结论。
- 如何把风味词变成可检索的长期记忆标签。

## 10. MVP 建议

第一版先做：

- 基础风味词库。
- 柑橘、花香、茶感、甜感、发酵、负面风味六类重点增强。
- 输入关键词后返回 5 到 8 个候选。
- 用户可接受、拒绝、编辑候选。
- 被接受的候选进入本次品鉴记录。

暂缓：

- 完整复刻 SCA 风味轮。
- 严格杯测评分体系。
- 自动判断风味真伪。
