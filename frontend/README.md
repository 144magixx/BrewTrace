# 前端说明

前端现在是 React + Vite + TypeScript 本地工作台。首页直接进入“当前记录”工作流，不是 landing page。

项目全貌、当前真实能力和文档导航见 [项目当前上下文](../docs/architecture/current-project-context.md)。本文只保留前端快速运行与验收信息。

## 安装依赖

```bash
npm install
```

## 常用命令

```bash
npm test
npm run build
npm run test:e2e
npm run dev
```

开发服务启动后监听：

```text
http://127.0.0.1:5173
```

Vite 已配置 `/api` 代理到 `http://127.0.0.1:8080`，因此本地验收时先启动后端，再启动前端。

## 当前模型运行方式

工作台只保留真实文本模型模式：

- `openai-gpt55` 是当前保留的历史模式代码，实际模型名不固定为 `gpt-5.5`。
- 后端每次启动前必须从 `$CODEX_HOME/config.toml` 或 `~/.codex/config.toml` 读取当前 `model` 和 Provider `base_url`，映射为 `TEXT_MODEL` 与 `OPENAI_BASE_URL`。
- 前端只展示后端快照返回的实际 `modelName` 和代理地址，不自行决定模型配置。
- 真实结构化输出可能需要 60-90 秒，本地默认 `MODEL_TIMEOUT_SECONDS=120`。

前端固定传递 `modelMode=openai-gpt55`，不会保存或展示密钥。右侧请求预览展示后端 HTTP Client 实际发送的请求体，只做敏感值脱敏，不通过前端或请求工厂重新构造 JSON。鉴权失败时页面显示可恢复模型错误，不生成本地替代文案。

## 验收入口

1. 点击“开始记录”创建会话。
2. 输入“今天喝了一支水洗埃塞，有柑橘和红茶感”。
3. 确认页面显示的模型名与后端 `/api/workbench/snapshot` 一致；鉴权失败时应展示可恢复错误。
4. 信息不足时确认模型返回一个自然追问；信息足够并返回 `POST` 时，确认出现克制版、夸张版和锐评版草稿。
5. 查看 Agent 状态区，确认完整会话上下文、事实边界、实际模型请求和模型响应可检查；完整工具 Trace 前端仍属于后续能力。
6. 若模型调用 `flavor_suggestion`，确认候选风味仍标记为待确认联想，不能直接写入已确认事实。
7. 点击“新建记录 / 清空当前会话”，先取消，确认状态不变；再次点击并确认，页面回到空记录，刷新后不恢复旧草稿。

## 边界说明

- `openai-gpt55` 是唯一文本模型模式。
- 当前未接真实长期数据库，候选记忆保持空状态。
- 当前主模型只注册了 `flavor_suggestion` 工具；图片生成和小红书工具没有进入主模型 Tool Calling 列表。
- 当前未执行小红书搜索、发布、评论、点赞或收藏动作。
- 清空当前会话只清空当前可见会话和浏览器恢复状态，不删除长期记忆、历史归档或外部平台数据。
