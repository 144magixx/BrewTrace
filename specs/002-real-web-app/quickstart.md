# Quickstart：真实 Web 应用接入验证指南

## 前置条件

- 本地已安装可运行 Java 21 及以上的 JDK。
- 本地已安装 Node.js 与 npm。
- 当前切片不要求 PostgreSQL、pgvector、Kafka、Redis、真实模型 API Key 或小红书登录态。
- 如果本机默认 `java` 指向 Java 8，后端命令必须显式使用 Homebrew OpenJDK 或其他 Java 21+ 路径。

## 自动化验证目标

本切片完成后，应能执行：

```bash
cd backend
./mvnw test

cd ../frontend
npm test
npm run build
npm run test:e2e
```

并补充真实服务验证：

```bash
cd backend
./mvnw spring-boot:run

cd ../frontend
npm run dev
```

前端启动后，应输出本地页面地址。用户在浏览器打开该地址，完成下方场景。

## 场景 1：打开真实工作台

1. 启动后端本地服务。
2. 启动前端本地页面。
3. 打开浏览器页面。

期望结果：

- 页面不是 landing page。
- 页面默认进入“当前记录”工作台。
- 页面可见“今天喝了什么咖啡？”输入入口。
- 页面可见当前记录摘要区域和右侧轨迹/状态预留区域。

## 场景 2：首轮对话追问

1. 创建新会话。
2. 输入：“今天喝了一支水洗埃塞，有柑橘和红茶感”。
3. 提交消息。

期望结果：

- 页面显示用户消息。
- 页面显示助手追问豆子信息、冲煮参数和文案风格。
- 当前记录摘要显示“水洗”“埃塞”“柑橘”“红茶”等已识别信息时，必须明确哪些仍待确认。
- 页面不展示最终文案草稿。

## 场景 3：补充事实生成三版草稿

1. 在同一会话中补充：“豆子是某烘焙商的埃塞水洗豆，水温 92 度，粉水比 1:15，想看克制、夸张和锐评。”
2. 提交消息。

期望结果：

- 页面展示克制版、夸张版和锐评版三类草稿。
- 每类草稿都有标题、正文和事实边界说明。
- 甜橙、青柠、葡萄柚等扩展词如果出现，必须标记为联想或待确认候选。
- 页面不得把未确认风味写成“我喝到甜橙”或同等事实表述。

## 场景 4：服务不可用错误

1. 停止后端服务。
2. 在前端输入一段咖啡描述并提交。

期望结果：

- 页面显示服务不可用或可重试错误。
- 输入框或恢复区域保留用户刚刚输入的内容。
- 页面给出检查本地服务或重试的建议。

## 场景 5：刷新恢复

1. 创建会话并输入至少一条消息。
2. 刷新浏览器页面。

期望结果：

- 页面尽量恢复最近会话或草稿输入。
- 如果无法恢复，页面明确提示需要重新创建会话。
- 页面不得静默清空用户输入并假装仍在原会话中。

## 场景 6：安全检查

执行以下检查：

```bash
rg -n "sk-[A-Za-z0-9_-]{20,}|AKIA[0-9A-Z]{16}|Authorization: Bearer|Cookie:|Session Token" .
```

期望结果：

- 不出现真实密钥或凭证。
- 仅允许命中文档中“不得记录”的说明文字或明显占位符。

## 已验证与未验证边界

完成本切片后已验证：

- 真实浏览器工作台可打开。
- 页面动作可到达本地服务并复用现有 Agent 业务内核。
- US1 对话文案主流程可见、可测、可演示。
- 错误提示和输入保留可见。

仍未验证：

- 真实数据库持久化。
- 真实模型调用。
- 小红书真实搜索和发布。
- 图片生成。
- 完整 AgentTrace SSE 侧边栏。
