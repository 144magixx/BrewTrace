## 概要

本功能把当前已验证的离线业务内核接成真实可打开的本地 Web 应用，优先交付 US1 对话文案垂直切片。用户应能打开本地浏览器工作台，创建品鉴会话，输入咖啡体验，看到助手追问；补充豆子、冲煮参数和文案风格后，页面展示克制版、夸张版和锐评版三类草稿，并持续区分用户确认事实与模型联想。

实现思路是最小化接入真实运行时：后端将现有会话、Agent 编排和文案业务内核暴露为本地服务；前端将当前离线字符串组件升级为可交互工作台页面；自动化与人工验收覆盖打开首页、创建会话、提交消息、展示追问、补充事实生成草稿、错误提示和事实边界展示。真实数据库、真实模型、小红书发布、生图、完整轨迹侧边栏暂不进入本切片，以避免把第一版真实 Web 验收变成过大范围。

主要风险包括：本机 Java 运行时版本不一致、前端依赖安装失败、请求失败时用户输入丢失、离线内核与真实页面状态同步不清，以及页面把联想词误展示成确认事实。

## 技术上下文

**语言/版本**：Java 21 作为后端目标运行基线；TypeScript 5.x 作为前端目标语言；当前本机默认 `java` 可能指向 Java 8，运行命令必须显式使用可满足 Java 21 及以上的 JDK。

**主要依赖**：后端接入 Spring Boot Web 运行时并复用现有 Maven 工程；前端接入 React、Vite 和浏览器测试工具；真实模型调用仍使用当前 Fake/离线适配器，不引入真实模型依赖作为默认验收条件。

**存储**：本切片默认使用内存会话状态和浏览器本地可恢复输入，不接入 PostgreSQL、pgvector、Redis 或 Kafka。持久化数据库留给后续独立切片。

**测试**：后端保留现有 Java 行为测试，并增加真实本地服务契约验证；前端增加组件/流程测试和浏览器验收；端到端验收覆盖用户可见流程和错误恢复。

**目标平台**：本地开发环境、桌面 Web 浏览器、本地后端服务、本地前端开发服务。

**项目类型**：前后端分离 Web 应用的真实运行时垂直切片。

**性能目标**：本地启动后，用户提交首轮消息 3 秒内看到追问或可恢复错误；用户补充事实后 3 秒内看到三类草稿；用户 10 分钟内能按 quickstart 完成首次浏览器验收。

**约束**：API Key、账号凭证、Cookie、Authorization Header 和 Session Token 不得进入 Git、页面、日志、测试输出或文档；本切片不得执行公开发布、点赞、评论、收藏或真实生图；页面不得把待确认联想写成用户事实。

**规模/范围**：MVP 本地单用户；只接通对话文案主流程、错误恢复、本地验收文档和最小工作台 UI。模板记录、长期记忆、真实轨迹侧边栏、发布包、小红书真实工具、数据库持久化和真实模型调用均不属于默认范围。
