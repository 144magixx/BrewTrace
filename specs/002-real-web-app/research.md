# Phase 0 调研：真实 Web 应用接入

## 调研范围

本文件解决“把离线业务内核接成真实 Web 应用”这一切片的关键范围、运行时、状态恢复、测试和安全边界问题。

## 1. 第一版范围

**Decision**：第一版只打通 US1 对话文案主流程：浏览器工作台、创建会话、提交消息、追问缺失事实、补充事实后生成三类草稿、页面展示事实边界和错误恢复。

**Rationale**：当前项目已经实现了较多离线内核，但用户无法从真实页面体验。若同时接数据库、真实模型、小红书、发布、生图和完整轨迹，会让第一版真实运行时风险过大。US1 是 Agent 产品主干，能最直接证明“业务内核可以通过页面真实使用”。

**Alternatives considered**：

- 一次性接入所有 MVP 能力：覆盖面完整，但依赖和调试面过大，不利于稳定验收。
- 只做后端真实服务：不能满足“真实 Web 应用”的用户体验。
- 只做前端静态页面：仍然无法验证页面动作到业务内核的真实链路。

## 2. 后端运行时策略

**Decision**：后端接入真实本地 Web 服务入口，复用现有 `TastingSessionApplicationService`、`AgentOrchestrator`、`DraftAgent` 和事实边界逻辑；第一版继续使用内存 Repository 和 Fake/离线模型适配器。

**Rationale**：这样可以用最小改动证明真实服务链路，同时保留已经验证的业务规则。数据库、真实模型、事务和安全配置可以在后续切片中逐个替换，不影响本次用户可见价值。

**Alternatives considered**：

- 直接引入数据库持久化：更接近最终形态，但会增加迁移、连接、测试容器和本地依赖问题。
- 直接接真实模型：体验更智能，但需要真实 Key、成本控制和网络稳定性，且不是证明 Web 链路的必要条件。
- 自写轻量 HTTP 服务：依赖少，但偏离项目默认技术栈和后续学习目标。

## 3. 前端工作台策略

**Decision**：前端把当前字符串组件升级为真实可交互工作台，保留三栏布局和“当前记录”主页面，优先实现对话输入、消息流、当前记录摘要、草稿 Tabs 和错误提示。

**Rationale**：用户需要在浏览器里完成核心流程，而不是看测试输出。三栏设计是已有设计约束，但本切片只需让中间主流程和当前记录摘要真实可用；右侧完整 Agent 轨迹栏可先显示预留/基础状态。

**Alternatives considered**：

- 完整实现所有导航页：工作量大，且与 US1 验收无关。
- 做营销页或说明页：不符合项目工作台定位，也不能验收真实业务流程。
- 只做简单表单：交互可用但无法承接既有工作台设计和 Agent 学习目标。

## 4. 状态恢复与错误处理

**Decision**：第一版使用本地单用户会话状态，页面刷新后尽量恢复最近会话标识和草稿输入；当无法恢复时，给出明确提示和重新创建路径。所有请求失败都必须保留输入框内容并显示可恢复建议。

**Rationale**：真实 Web 应用的初期常见失败是服务没启动、端口冲突或刷新丢状态。保留输入和明确恢复路径能降低验收挫败感，也符合“用户数据不丢”的项目原则。

**Alternatives considered**：

- 每次刷新都清空状态：实现简单，但用户体验差，容易造成误判。
- 第一版就做后端持久化恢复：更稳，但增加数据库依赖，不符合本切片范围。
- 静默重试：可能掩盖服务状态，且不利于用户学习系统链路。

## 5. 测试与验收策略

**Decision**：保留现有后端行为测试和前端脚本测试；新增真实服务契约验证、前端流程测试和浏览器人工验收。真实模型、真实数据库和小红书账号不作为本切片默认验收条件。

**Rationale**：本切片的价值在“真实页面到本地服务到 Agent 内核”的链路，因此必须有浏览器验收；同时离线自动化测试继续保护事实边界和业务规则。

**Alternatives considered**：

- 只做人工验收：容易回归。
- 只做自动化测试：不能证明页面真实可用。
- 把真实外部依赖纳入默认验收：验收不稳定，也会引入凭证和成本风险。

## 6. 安全边界

**Decision**：本切片不执行公开发布、点赞、评论、收藏或真实生图；不需要真实模型 Key。若本地环境存在 Key，也不得显示在页面、日志、测试输出或文档中。

**Rationale**：Web 接入切片不需要高影响工具。保持工具安全边界能避免“为了演示页面”误触发外部平台或成本动作。

**Alternatives considered**：

- 顺手接小红书发布：违反最小切片原则，且需要二次确认和人工验证。
- 顺手接真实生图：会引入成本动作，且不是 US1 验收必须。
- 在页面显示完整配置用于调试：学习上直观，但会造成凭证泄露风险。
