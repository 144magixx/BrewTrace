## 用户场景与测试 *(必填)*

### 用户故事 1 - 无密钥时给出可恢复错误（优先级：P1）

作为项目作者，我希望在没有配置模型密钥时，系统不要生成本地替代文案，而是保留用户输入和当前会话，显示可恢复错误，并提示检查本地密钥或重试，避免把非真实模型输出误认为 GPT-5.5 结果。

**为什么是这个优先级**：GPT-5.5 是唯一生成链路；无密钥时必须失败得清楚，不能用本地预设内容掩盖真实模型未调用。

**独立测试**：在未提供本地模型密钥的环境中打开工作台，输入一段咖啡体验，系统应展示 `MODEL_AUTH_FAILED` 或等价可恢复错误，保留输入和会话状态，且不出现三版本地替代文案。

**验收场景**：

1. **假如** 本地没有可用模型密钥，**当** 用户输入咖啡体验，**那么** 系统返回可恢复模型错误，不生成本地替代文案。
2. **假如** 用户查看模型输出区域，**当** 真实模型未成功返回，**那么** 页面不得显示三版草稿。
3. **假如** 用户查看上下文预览，**当** 当前调用失败，**那么** 页面仍展示将发送给 GPT-5.5 的上下文组织结果，并保留重试入口。

---

### 用户故事 2 - 启用真实 GPT-5.5 生成三版文案（优先级：P1）

作为项目作者，我希望在本地配置模型密钥并启用 `openai-gpt55` 模式后，输入一段咖啡体验即可获得真实模型生成的三版小红书文案：克制版、夸张版、锐评版，并能看到真实模型状态、模型名称和生成结果来源。

**为什么是这个优先级**：这是本功能的核心垂直切片，证明工作台已收敛为可配置真实模型链路，同时保持用户可见状态清楚。

**独立测试**：在本地配置有效密钥并选择真实模型模式后，输入一段咖啡体验；系统应调用真实模型生成三版文案，并在页面显示“真实模型输出 / GPT-5.5”或含义等同的状态。

**验收场景**：

1. **假如** 本地存在有效模型密钥且用户启用 `openai-gpt55`，**当** 用户输入“今天喝了一支水洗埃塞，酸质像柑橘，尾段有红茶感”，**那么** 系统返回克制版、夸张版和锐评版三版文案。
2. **假如** 真实模型生成完成，**当** 用户查看模型输出区域，**那么** 页面显示真实模型状态、模型名称、生成时间或最新生成状态，以及三版结果。
3. **假如** 用户重复提交，**当** 再次生成文案，**那么** 页面必须清楚展示最新一次 GPT-5.5 输出，不得复用上一次结果冒充新输出。

### 用户故事 3 - 预览真实模型请求边界（优先级：P1）

作为项目作者，我希望在调用真实模型前后，前端上下文预览能清楚展示将发送给模型的内容、不会发送的内容、已经发送的真实请求和收到的模型响应，并用中文标签区分“将发送”“不会发送”“已发送给大模型”“大模型返回”。

**为什么是这个优先级**：项目宪法要求可追踪的 Agent 状态。真实模型接入后，用户必须能看到上下文如何被组织，确认未确认联想、候选记忆和页面辅助信息没有被错误地发送或写成事实。

**独立测试**：输入咖啡体验后，用户无需阅读日志即可在工作台右侧预览模型上下文；真实模型模式下还能看到原始请求 JSON 的代码框展示，以及模型返回内容的独立标签。

**验收场景**：

1. **假如** 当前会话包含用户输入、已确认事实、待确认联想和候选记忆边界，**当** 用户查看上下文预览，**那么** 页面按中文标签区分会发送内容和不会发送内容。
2. **假如** 用户启用真实模型模式并触发生成，**当** 请求被发送，**那么** 前端展示发送给大模型的真实请求，并以类似 Markdown 的 JSON 显示框呈现原始结构。
3. **假如** 模型返回结果，**当** 用户查看上下文预览和输出区域，**那么** 发送内容与收到内容必须使用不同中文标签，不得混在同一段文本中。
4. **假如** 某条候选记忆或待确认联想未被纳入模型输入，**当** 用户查看预览，**那么** 页面必须显示它不会发送或需确认后才可发送。

### 用户故事 4 - 保护事实边界与输出可信度（优先级：P1）

作为项目作者，我希望真实模型输出也必须遵守咖啡事实边界：区分用户确认事实、模型推断和待用户确认联想，不能把未确认风味、产区、处理法、偏好或候选记忆写成事实，让最终文案既有创作表达也不伪造咖啡记录。

**为什么是这个优先级**：真实模型会带来更强创作能力，也更容易产生看似合理但未被确认的表达。事实边界检查是本项目最重要的可信度防线。

**独立测试**：使用包含少量已确认事实和若干待确认联想的样例输入生成文案；系统应在输出区域展示事实边界检查结果，指出哪些表达有用户依据、哪些属于模型推断、哪些需要用户确认。

**验收场景**：

1. **假如** 用户只确认“柑橘感”，**当** 模型输出写到“像甜橙爆汁”，**那么** 系统必须把该表达标为模型推断或待确认联想，不得列为用户确认事实。
2. **假如** 模型输出引用用户明确输入的“水洗埃塞”或“红茶感”，**当** 用户查看事实边界检查，**那么** 检查结果应指向对应用户输入来源。
3. **假如** 模型输出新增未确认的豆庄、海拔、烘焙商、处理法或用户偏好，**当** 用户查看检查结果，**那么** 系统必须将其标为无依据或需用户确认。
4. **假如** 三版文案风格不同，**当** 用户查看检查结果，**那么** 每一版都必须保持事实、推断和待确认联想的边界可见。

### 用户故事 5 - 模型失败后可恢复重试（优先级：P2）

作为项目作者，我希望当真实模型调用出现超时、鉴权失败、限流或返回格式异常时，页面展示可理解、可恢复的错误，并保留我的输入、当前会话、上下文预览和已生成状态，让我能调整配置、切换模式或重试。

**为什么是这个优先级**：真实模型依赖外部服务，失败是常态。可恢复错误能避免用户丢失咖啡记录，也能让本地开发排查更可靠。

**独立测试**：分别模拟超时、鉴权失败、限流和格式异常；系统应显示对应错误类型、建议动作和重试入口，并保留用户输入与当前会话状态。

**验收场景**：

1. **假如** 真实模型调用超时，**当** 页面显示错误，**那么** 用户能看到超时说明、重试入口，并且原始咖啡输入仍保留。
2. **假如** 鉴权失败或密钥不可用，**当** 用户查看错误，**那么** 页面提示检查本地密钥或切换离线模式，不展示任何密钥内容。
3. **假如** 服务限流，**当** 用户查看错误，**那么** 页面说明可以稍后重试或切换离线模式，并保留当前会话状态。
4. **假如** 模型返回格式异常，**当** 用户查看错误，**那么** 页面说明生成结果暂不可用，并保留可检查的上下文预览和重试入口。

### 用户故事 6 - 密钥与敏感信息不泄露（优先级：P2）

作为项目作者，我希望真实模型接入后，密钥、鉴权头、Cookie 和会话令牌只留在本地安全边界内，不进入配置文件、日志、测试快照、前端页面或错误消息，避免把模型凭证和会话信息暴露到仓库或验收材料中。

**为什么是这个优先级**：模型凭证涉及成本和账号安全，且项目宪法明确要求 API Key 不得写入 Git、日志、截图或生成文档。

**独立测试**：运行相关测试并检查前端页面、错误消息、日志输出和测试快照，确认不会出现 API Key、Authorization、Cookie 或 Session Token。

**验收场景**：

1. **假如** 本地配置了模型密钥，**当** 用户查看前端页面和上下文预览，**那么** 页面不得展示密钥、鉴权头或任何可复用凭证。
2. **假如** 真实模型调用失败，**当** 系统记录错误或展示错误，**那么** 日志和错误消息不得包含 API Key、Authorization、Cookie 或 Session Token。
3. **假如** 自动化测试生成快照或断言输出，**当** 用户检查测试产物，**那么** 不得出现真实密钥或可复用凭证。

### 边界情况

- 当本地没有模型密钥时，系统必须给出可恢复错误，不得生成本地替代文案。
- 当用户启用真实模型模式但密钥不可用或无效时，系统必须给出可恢复错误，不得吞掉用户输入。
- 当真实模型返回内容缺少任一指定版本时，系统必须把结果标为格式异常或部分不可用，并支持重试。
- 当真实模型输出包含未确认咖啡事实时，系统必须在事实边界检查中标出，不得自动归入确认事实。
- 当待确认联想、候选记忆或页面辅助说明未进入请求时，预览必须明确显示其未发送原因。
- 当真实请求 JSON 展示在前端时，系统必须去除密钥、鉴权头、Cookie、会话令牌和其他可复用凭证。
- 当用户在失败后重试时，系统必须复用当前会话和可见上下文，不要求用户重新输入咖啡记录。
- 当用户重试模型调用时，系统必须清楚区分最新输出和历史输出，避免把旧结果标成新结果。
- 当小红书发布、长期记忆数据库、外部工具执行或自动发布动作尚未接入时，页面只能显示未接入或未执行状态，不得暗示已经发生外部动作。

## 需求 *(必填)*

### 功能需求

- **FR-001**：系统必须只支持 `openai-gpt55` 一种模型模式，并在工作台中清楚显示当前模式和输出来源。
- **FR-002**：在没有本地模型密钥时，系统必须返回可恢复模型错误，不得生成本地替代文案。
- **FR-003**：当真实模型未成功返回时，系统不得在模型状态和输出区域显示三版草稿。
- **FR-004**：当本地密钥可用且用户提交咖啡记录时，系统必须调用真实 GPT-5.5 生成结果，并在页面显示真实模型状态和模型名称。
- **FR-005**：真实模型生成结果必须包含三版小红书文案：克制版、夸张版、锐评版。
- **FR-006**：系统必须保留清晰的模型调用边界，使业务链路能够通过可替换模型网关获得模型结果，而不把密钥、第三方调用细节或传输细节暴露给前端和普通业务状态。
- **FR-007**：系统必须只从本地运行环境读取模型密钥，不得要求用户把密钥写入项目配置文件、前端代码、测试数据或可提交文档。
- **FR-008**：系统必须继续沿用当前工作台、右侧 Agent 状态卡片、上下文预览和事实边界面板，不得用全新页面替代现有主流程。
- **FR-009**：用户输入咖啡记录后，系统必须组织当前会话、已确认事实、待确认联想和候选记忆边界，形成可预览的模型上下文。
- **FR-010**：上下文预览必须用中文标签区分将发送给模型的内容、不会发送的内容、待确认后才可能发送的内容、已发送给大模型的内容和大模型返回的内容。
- **FR-011**：真实模型模式下，前端必须以类似 Markdown 的 JSON 显示框展示发送给大模型的原始请求结构，并确保其中不包含密钥、鉴权头、Cookie 或会话令牌。
- **FR-012**：系统必须在模型输出区域显示真实模型状态、模型名称、三版生成结果和事实边界检查结果。
- **FR-013**：事实边界检查必须区分用户确认事实、模型推断、待用户确认联想、候选记忆来源和无依据表达。
- **FR-014**：模型输出不得把未确认风味、产区、处理法、烘焙信息、用户偏好或候选记忆写成用户确认事实。
- **FR-015**：当真实模型调用超时、鉴权失败、限流或返回格式异常时，系统必须展示可恢复错误、建议动作和重试入口。
- **FR-016**：模型调用失败后，系统必须保留用户输入、当前会话状态、上下文预览和可见状态卡片。
- **FR-017**：系统必须允许用户在真实模型失败后重试，不得提供本地替代生成入口。
- **FR-018**：系统不得在本次功能中执行真实小红书发布、真实长期记忆数据库写入或召回、外部工具执行、自动发布动作或其他高影响外部动作。
- **FR-019**：系统必须确保日志、错误消息、测试快照和前端页面不包含 API Key、Authorization、Cookie、Session Token 或其他可复用凭证。
- **FR-020**：系统必须提供可验证样例或测试路径，覆盖无密钥可恢复错误、有效密钥真实生成、上下文预览、三版输出、事实边界、错误恢复和敏感信息不泄露。

### 宪法对齐 *(必填)*

- **真实性**：真实模型输出必须区分用户确认事实、模型推断、待确认联想、候选记忆和无依据表达；未确认内容不得被写成咖啡事实。
- **可追踪性**：本功能必须展示当前模型模式、模型状态、模型名称、上下文预览、真实请求摘要、模型返回、三版文案和事实边界检查结果，并说明哪些内容会影响模型输出。
- **工具安全**：模型密钥只能来自本地运行环境；API Key、Authorization、Cookie 和 Session Token 不得进入页面、日志、测试快照或文档。本功能不执行小红书发布、真实长期记忆数据库、外部工具调用或自动发布动作。
- **验证方式**：必须通过自动化测试或人工验收覆盖无密钥可恢复错误、真实模型生成、上下文预览、原始 JSON 展示、三版输出、事实边界检查、超时、鉴权失败、限流、格式异常、重试和敏感信息过滤。
- **学习价值**：本功能帮助用户学习真实模型链路中的模型网关边界、上下文组织、请求/响应可视化、输出结构化、错误恢复和事实边界控制。
- **中文文档**：本功能相关规格、计划、任务、调研、验收说明和学习材料默认使用简体中文。

### 关键实体 *(涉及数据时填写)*

- **模型模式**：表示当前生成链路的来源，当前只包含真实 GPT-5.5 模式，以及当前可用性和展示标签。
- **模型上下文包**：表示一次生成前组织出的上下文，包含当前会话、已确认事实、待确认联想、候选记忆边界、发送状态和排除原因。
- **真实请求预览**：表示前端可见的原始请求结构展示，包含将发送给模型的业务内容和必要元信息，但不包含任何可复用凭证。
- **模型响应快照**：表示一次生成返回的结果，包含模式、模型名称、状态、三版文案、返回时间或失败原因。
- **文案版本**：表示模型生成的一版小红书文案，包含版本类型、正文、所用事实、推断表达和待确认表达。
- **事实边界检查结果**：表示对每版文案的可信度检查，包含表达片段、依据类型、风险等级、来源指向和建议动作。
- **可恢复模型错误**：表示真实模型调用失败后的用户可见状态，包含错误类别、说明、建议动作、是否可重试和保留的会话状态。
- **敏感信息过滤记录**：表示系统对页面展示、日志和测试输出中的凭证泄露风险进行过滤或验证的结果。

## 成功标准 *(必填)*

### 可衡量结果

- **SC-001**：在无本地模型密钥的环境中，用户提交咖啡记录后 100% 得到可恢复模型错误，且模型输出区域不出现三版本地替代文案。
- **SC-002**：在配置有效本地密钥并启用 `openai-gpt55` 后，用户输入一段咖啡体验，系统能够在一次生成流程中返回克制版、夸张版、锐评版三版文案，并显示真实模型状态和模型名称。
- **SC-003**：上下文预览中所有发送与不发送内容都必须有中文标签；验收样例中用户能在 1 分钟内指出哪些内容已发送给大模型、哪些没有发送。

## 假设

- 第一阶段目标用户是项目作者本人，本地单用户使用，不新增登录、注册、多用户权限或远程部署要求。
- `openai-gpt55` 是唯一文本模型模式；无密钥和本地测试场景必须走可恢复错误或模型客户端 stub，不得生成本地替代文案。
- 用户明确要求本次真实模型模式面向 GPT-5.5，并在后续计划阶段使用 OpenAI Responses API 调用 `model: gpt-5.5`；规格阶段只定义用户可见行为、安全边界和验收结果。

- 本地模型密钥由运行环境提供；项目配置文件、前端、测试快照和文档不保存真实密钥。
- 当前工作台、右侧 Agent 状态卡片、上下文预览和事实边界面板已经存在或正在上一切片中落地，本功能在其基础上升级模型来源和请求/响应可视化。
