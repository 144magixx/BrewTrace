# Quickstart：模型驱动消息路由

## 目标

验证工作台能够根据模型结构化返回分流：

- 信息不足时进入 CONVERSATION，只在聊天框展示 `talk`，继续等待用户补充。
- 信息完备或用户明确要求生成时进入 POST，只在聊天框展示 `talk`，并在草稿区展示三版文案。
- 异常模型返回不会进入错误链路之外的业务流程。

## 前置条件

- 已完成 spec5 的实现任务。
- 本地后端和前端依赖已安装。
- 如需真实模型验证，本地环境已配置模型 Key 和 base URL，且凭证不写入仓库。

## 启动

从仓库根目录运行后端测试：

```bash
cd backend
./mvnw test
```

启动后端：

```bash
cd backend
./mvnw spring-boot:run
```

启动前端：

```bash
cd frontend
npm test
npm run dev
```

打开本地前端页面后，进入咖啡 Agent 工作台。

## 场景 1：信息不足进入 CONVERSATION

输入：

```text
今天喝了一杯咖啡
```

期望：

- 工作台状态保持等待用户补充。
- 聊天框出现助手消息，内容来自 `talk`。
- `talk` 提出 1-3 个补充问题，例如风味、产区、处理法、冲煮参数。
- 草稿区为空。
- 不展示一键发送或发布确认入口。
- Agent 状态区显示模型决策为 CONVERSATION。

## 场景 2：非咖啡输入不伪造咖啡事实

输入：

```text
我喝了杯可乐
```

期望：

- 系统不生成咖啡豆、产区、处理法、海拔或风味事实。
- 聊天框只展示 `talk`，说明当前记录不足以生成真实咖啡品鉴文案，或询问是否要改为饮品记录。
- 草稿区为空，或只在用户明确要求“就按饮品记录生成”后生成带风险提醒的 POST。

## 场景 3：信息完备进入 POST

输入：

```text
今天喝了一支水洗埃塞，橙柑和红茶感明显，口感干净，温度下来茶感更强。没有更多补充了，给我生成文案吧。
```

期望：

- 工作台状态进入草稿可见。
- 聊天框只展示 `talk`，例如“信息已经够了，我先整理成三版文案”。
- 草稿区展示三版：克制版、夸张版、锐评版。
- 三版风格不重复、不缺失。
- 事实依据只引用用户已说过或已确认的内容。
- 发布相关入口仍需要发布前确认，不自动公开发布。

## 场景 4：POST 信息有限但用户明确要求生成

输入：

```text
只记得是水洗埃塞，别问了，直接生成吧。
```

期望：

- 系统可以进入 POST。
- 文案保持克制，不补写未提供的豆庄、海拔、烘焙商、品种或具体风味。
- `talk` 或风险提示说明信息有限。
- 事实边界检查能看到缺失信息或低置信度提醒。

## 场景 5：异常模型返回被阻断

模拟模型返回以下任一异常：

- 缺少 `talk`。
- `messageType` 不是 POST 或 CONVERSATION。
- POST 缺少任一风格文案。
- CONVERSATION 同时携带可发布草稿。

期望：

- 系统不进入草稿或发布链路。
- 前端展示可恢复错误。
- 当前用户输入和会话状态保留。
- 请求预览和响应预览不包含凭证。

## 场景 6：提示词模板检查

检查项：

- 模型路由提示词位于版本化 prompt 目录。
- POST/CONVERSATION 字段含义、`talk` 展示规则和事实边界规则来自模板。
- 风格提示词内容被实际组合进模型输入，而不是只出现文件路径。
- 运行代码和测试代码不包含成段系统提示词硬编码。

通过标准：

- 自动测试覆盖模板加载和动态占位符替换。
- 请求预览能说明模板版本或展示脱敏后的组合结果。
- 敏感信息扫描不发现 API Key、Authorization、Cookie 或 Session Token。

## 实现验证记录（2026-06-30）

已验证：

- 后端 `./mvnw test` 通过，覆盖模型消息解析、请求 schema、prompt 动态组合、POST/CONVERSATION 工作台路由和事实边界。
- 前端 `npm test` 通过，覆盖聊天框只展示 `talk`、POST 草稿展示、模型输出面板和 Agent 状态卡片。
- prompt 模板已位于 `backend/src/main/resources/prompts/`，三种风格提示词正文会被 `PromptComposer` 动态组合进请求。

未验证：

- 未使用真实 API Key 调用外部模型做在线端到端验证。
- 未启动本地浏览器执行人工点击流程。

残余风险：

- 真实模型仍可能返回语义上不够理想的 `talk` 或 warnings；当前后端已校验结构和关键路由边界，但内容质量仍需继续通过样本迭代 prompt。
